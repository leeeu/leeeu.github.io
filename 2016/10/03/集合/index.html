集合

发表于 2016-10-03   |   分类于 JavaSE

Collection是一个接口，定义了集合的相关的操作方法，有两个子接口，分别为List和Set，集合中存储的都是引用类型元素，只保存每个元素对象的引用。
接口不能实例化，只能通过引用类型变量指向接口实现类的实例化对象。

Collection常见的方法有如下几个：

boolean add(E e) //将给定元素添加到集合中，成功返回true，否则为false  
boolean contains(Object o)// 如果此集合包含指定的元素，则返回 true，通常有必要重写equals()方法  
int size() //返回此集合中的元素数   
void clear() //移除此集合中的所有元素   
boolean isEmpty() //如果此集合不包含元素，则返回 true  
boolean addAll(Collection<? extends E> c) //将指定集合中的所有元素都添加到此集合中   
boolean containsAll(Collection<?> c) //如果此集合包含指定集合中的所有元素，则返回 true

Collection定义了一个返回迭代器的方法：

Iterator<E> iterator() // 返回在此集合的元素上进行迭代的迭代器

其中Iterator是一个接口，遍历的时候要遵循“问、取、删（非必要）”原则，即先调
boolean hasNext() //判断集合是否还有元素可以遍历，
一般在while()循环中调用，接着取元 素E next() //返回迭代的下一个元素，删除操作不是非必要的，如果要删除元素，必须要用迭代器删除，不能用集合删除，删除方法为void remove()
在JDK5.0之后推出了增强for循环，方便的遍历集合或数组,，语法如下：

for(元素类型 e : 集合或数组) {
    循环体
}
  • List:可存储重复元素，常用实现类有ArrayList、LinkedList，分别用动态数组和链表实现List，前者更适合随机访问，而后者更适合插入删除

List除了继承Collection外还定义了一系列方法，常用的有：

E get(int index) //返回列表中指定位置的元素
E set(int index, E element) //用指定元素替换列表中指定位置的元素 
boolean add(E e) //向列表的尾部添加指定的元素 
E remove(int index) //移除列表中指定位置的元素 
List<E> subList(int fromIndex, int toIndex) //返回列表中指定的 fromIndex（包括 ）和 toIndex（不包括）之间的部分，修改子集合会影响原集合
<T> T[] toArray(T[] a) //返回包含此集合中所有元素的数组，数组的类型与指定数组的类型相同

可以将数组转为List集合：

static <T> List<T> asList(T... a) //返回一个受指定数组支持的固定大小的列表，对集合的修改会影响数组，不能对集合进行增删元素

List的排序：

static <T extends Comparable<? super T>> void sort(List<T> list) //根据元素的自然顺序 对指定列表按升序进行排序，需要实现Comparable接口并重写int comparaTo(T t)方法
static <T> void sort(List<T> list, Comparator<? super T> c) //根据指定比较器产生的顺序对指定列表进行排序，需要实现Comparator接口并重写int compare(T o1,To2)
  • Set:不能存储重复元素，常用实现类有HashSet、LinkedHashSet

队列和栈

  • Queue队列，遵循先进先出FIFO，只能从一段添加元素，从另一端取出元素，LinkedList实现该接口

主要方法如下：

boolean offer(E e); //讲一个对象添加到队尾，成功返回true
E poll(); //从队首删除并返回一个对象
E peek(); //返回队首元素（不删除）
  • Deque是Queue的子接口，可以从队列两端入队和出队，LinkedList实现该接口，如果只在一端入队和出队，即是栈，栈遵循先进后出，入栈push，出栈pop

Map接口

  • Map接口定义的集合又成查找表，用于存储键值对。
  • 常用实现类为：HashMap和TreeMap

常用方法：

V put(K key, V value) //将指定的值与此映射中的指定键存放到Map中，如果key已存在则会替换对应的value
V get(Object key) //返回指定键所映射的值；如果此映射不包含该键的映射关系，则返回 null 
boolean containsKey(Object key) //如果此映射包含指定键的映射关系，则返回 true

HashMap

  • 存放入HashMap的key必须重写hashCode和equals方法

遍历Map

Set<K> keySet() //返回此映射中包含的键的 Set 视图,通过get(key)获取对应值
Set<Map.Entry<K,V>> entrySet() //返回此映射中包含的映射关系的 Set 视图，通过Entry接口的 K getKey()和 V getValue()返回对应的键值

有序的Map：LinkedHashMap
