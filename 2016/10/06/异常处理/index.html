异常处理

发表于 2016-10-06   |   分类于 JavaSE

Java异常中定义有Throwable类，由这个类派生出Exception和Error两个子类。其中Exception表示由于网络故障，文件损坏等情况导致的异常；而Error表示Java运行时环境出现的错误，比如JVM内存资源耗尽等。

语法如下：

try {
    //该段代码可能会产生并抛出一种或几种类型的异常对象
} catch(someException e) {
    //如果捕获到异常在这里做相应的处理，如果没有异常则忽略这段代码
}

一个try语句块可以伴随一个或者多个catch语句，用于处理不同类型的异常。catch捕获的异常类型从上到下的顺序应该是从子类到父类。
finally{…}语句不单独存在，可以是 try-finally 或者是 try-catch-finally。
无论 try 语句中是否抛出异常，finally 中的代码都是要执行，即使 try 语句中已经有 return 了，除非在执行 finally 前执行了 System.exit 的方法。
通常在 finally 中进行资源的释放工作。

throw关键字
当程序发生异常而无法处理时，会抛出对应的异常对象。除此之外，我们也能自行抛出异常，让调用者来处理这个异常。例如 throw new ArithmeticException();

throws关键字
当方法可能会因为某些错误抛出异常时，使用 throws 声明方法将会抛出的异常，以便提醒调用者做出相应的处理。
在使用继承时，重写父类方法中有声明 throws 某些异常，子类可以不声明 throws 异常、可以仅throws 父类中声明的异常、也可以 throws 父类声明的异常的子类异常。 不可以 throws 出额外异常、或者 throws 出父类声明的异常的父类异常。

异常可以分为检查异常和非检查异常：

  • 检查异常：编译器会强制执行处理或声明规则，不捕获这个异常则不能通过编译。
  • 非检查异常：在产生异常时，不一定非要采取任何操作，编译器不会检查是否已经解决了异常。

RuntimeException类属于非检查异常，此类异常会在程序中频繁出现，所以不受编译器检查与处理或声明规则的限制。常见的RuntimeException有：

  • IllegalArgumentException
  • NullPointerException
  • ArrayIndexOutOfBoundsException
  • ClassCastException
  • NumberFormatException

Exception常用API

void printStackTrace() //用来跟踪异常事件发生时执行堆栈的内容
String getMessage() //获得有关异常事件的信息
Throwable getCause() //获取该异常出现的原因

自定义Exception

有时候为了更精准的捕获和处理异常已呈现更好的用户体验，需要自定义异常。
创建语法如下：

class exceptionName extends Exception {
    ...
}

定义好异常类后使用Eclipse自动生成相应构造方法即可。
