多线程

发表于 2016-10-07   |   分类于 JavaSE

进程是操作系统中运行的一个任务，一个应用程序运行在一个进程中。一个进程中包含一个或者多个线程，当操作系统创建一个进程后，会自动申请一个名为主线程或首要线程的线程。
同一个进程中的线程共享一块内存空间和一组系统资源，线程之间的切换负荷小。

创建线程

通常使用匿名内部类的方式创建线程，以减少代码复杂度。

  • 可以通过继承Thread类并重写run()方法来定义一个线程。

注意：启动线程是调用线程的start()方法而不是直接调用run()方法，start方法将线程纳入线程调度，当线程获得时间片后会自动开始执行run方法。
  • 实现Runnable接口并重写run()方法来定义线程体，然后在创建线程的时候将Runnable的实例传入并启动线程。

这样可以将线程和任务分开减少耦合度，同时Java是单继承，而可以实现多个接口，不影响继承其他父类。

线程操作API

线程优先级

线程优先级划分为1-10，1最小，10最大，默认为5.

设置优先级方法如下：

void setPriority(int newPriority) //更改线程的优先级

常用常量：

static int MAX_PRIORITY //线程可以具有的最高优先级 
static int MIN_PRIORITY //线程可以具有的最低优先级 
static int NORM_PRIORITY //分配给线程的默认优先级

守护线程

当进程中的前台线程都结束了，守护线程会强制终止。创建出来的线程默认是前台线程，设置守护线程方法如下：

void setDaemon(boolean on) //将该线程标记为守护线程或用户线程 

static void sleep(long millis) //在指定的毫秒数内让当前正在执行的线程休眠，释放CPU不释放锁
static void yield() //暂停当前正在执行的线程对象，并执行其他线程 
void join() //用于等待当前线程终止

线程同步

多个线程并发读写同一个临界资源会发生“线程并发安全问题”。
常见临界资源：

  • 多线程共享实例变量
  • 多线程共享静态公共变量

synchronized 关键字是Java中的同步锁
同步代码块：

若方法所有代码都需要同步，可以直接给方法加锁。应尽量减少锁的范围来提高并发效率。
通常使用this来作为锁对象。
静态方法与非静态方法同时声明synchronized，他们是非互斥关系的。应为静态方法锁的是类对象而非静态方法锁的是当前方法所属对象。

线程池

ExecutorService是Java提供用于管理线程池的类。
线程池的两个主要作用：

  • 控制线程数量
  • 重用线程

Executors 类提供了用于此包中所提供的执行程序服务的工厂方法

static ExecutorService newCachedThreadPool() //创建一个可根据需要创建新线程的线程池，但是在以前构造的线程可用时将重用它们 
static ExecutorService newFixedThreadPool(int nThreads) //创建一个可重用固定线程数的线程池，以共享的无界队列方式来运行这些线程 
static ScheduledExecutorService newScheduledThreadPool(int corePoolSize) //创建一个线程池，它可安排在给定延迟后运行命令或者定期地执行 
static ExecutorService newSingleThreadExecutor() //创建一个使用单个 worker 线程的 Executor，以无界队列方式来运行该线程

BlockingQueue接口

双缓冲队列，保证并发安全的同时提高队列的存取效率，方法同Queue。
常用实现类：

  • ArrayBlockingQueue：一个由数组支持的有界阻塞队列。此队列按 FIFO（先进先出）原则对元素进行排序。通过构造函数int参数规定大小
  • LinkedBlockingQueue：此队列按 FIFO（先进先出）排序元素。若构造函数带有int参数，则大小为该参数大小，不带参数则大小为Integer.MAX_VALUE
  • PriorityBlockingQueue：类似于LinkedBlockingQueue但是排序不是FIFO，而是对象的自然排序或者构造函数的Comparator决定
  • SynchronousQueue：一种阻塞队列，其中每个插入操作必须等待另一个线程的对应移除操作 ，反之亦然。即放和取必须是交替完成的
