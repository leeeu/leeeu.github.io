Oracle基础查询

发表于 2016-11-07   |   分类于 Oracle

基本查询语句

FROM子句

SELECT */列名,... FROM table_name;

SELECT：用于指定要查询的列，*表示查询所有列
FROM：指定查询的表

使用别名

使用列的别，在查询的列后面加上别名，用空格或者 AS 隔开，如果希望别名中区分大小写或者有空格，需要用双引号括起来。

WHERE子句

在SELECT语句中，可以在WHERE子句中使用比较操作符限制查询结果，如果是和字符串或者日期类型比较，需要用单引号括起来

SELECT子句

如果只查询表的部分列，在SELECT后指定列名

查询条件

  • 不等于使用 <> 其他比较符用法一样
  • AND：逻辑与
  • OR：逻辑或
  • LIKE：用于模糊查询，其中通配符 % 表示0到多个字符， _ 表示单个字符
  • IN(list)：取出符合列表范围中的数据
  • NOT IN (list)：取出不符合列表范围中的数据
  • BETWEEN…AND…：查询符合某个范围的数据
  • IS NULL/IS NOT NULL：用于判断是否为空值
  • ALL/ANY：不能单独使用，必须和比较操作符一起使用
  • DISTINCT：去除重复列

排序

使用 ORDER BY 子句，必须出现在SELECT中的最后一句，用 ASC 来指定升序排序，默认为升序， DESC 来指定降序排序，NULL值视为最大。
当以多列作为排序时，按列的次序进行排序，每一列需要单独设置排序方式。

聚合函数

  • MAX：取得列或表达式的最大值
  • MIN：取得列或表达式的最小值
  • AVG：统计列或表达式的平均值
  • SUM：统计列或表达式的和值
  • COUNT：计算表中的记录条数

聚合函数会忽略NULL值，如果需要统计可以使用空值函数NVL对相应的字段进行处理。

分组

GROUP BY 子句根据分组的条件将表进行分组，每组返回一个结果。
HAVING 子句用来对分组后的结果进一步限制，必须跟在GROUP BY后面，不能单独使用。

查询语句执行顺序

  1. FROM：执行顺序为从后往前，从右到左，数据量少的表尽量放在后面
  2. WHERE：执行顺序为从下到上，从右到左，能过滤大量记录的条件写在WHERE最右
  3. GROUP BY：从左到右分组，最好在分组前使用WHERE将不需要的记录过滤掉
  4. HAVING：尽量避免使用，HAVING会对所有记录检索一遍后才对结果集进行过滤，需要排序等操作，消耗资源
  5. SELECT：少用*，尽量用字段名称
  6. ORDER BY：执行顺序为从左到右，消耗资源

关联查询

查询两个或两个以上的数据表或视图的查询叫做连接查询。
笛卡尔积是指关联操作中每个表中的每一行数据都和其他表的每一行做组合，返回 M*N 条记录。

内连接

table_1 JOIN table_2
ON condition

不满足连接条件的记录不会出现在结果集中

外连接

外连接进行关联查询时除了会返回满足连接条件的记录，也会返回不满足连接条件的记录 

table_1 LEFT|RIGHT|FULL JOIN table_2
ON condition
  • 左外连接：以JOIN左侧表作为驱动表，该表所有记录都显示出来，当不满足条件时，该记录中来自JOIN右表中的全部字段为NULL
  • 右外连接：以JOIN右侧表作为驱动表，该表所有记录都显示出来，当不满足条件时，该记录中来自JOIN左表中的全部字段为NULL
  • 全外连接：是左外连接和右外连接的查询结果的总和

自连接

保存的数据是当前表的一条记录，可以对应当前表的多条记录，用于解决记录属性相同但记录间又存在上下级关系的树状结构时使用，通过将表用别名虚拟成两个表的方式实现。
